# Comparisons

MagAgent is best understood as a local-first, provider-agnostic coding and productivity agent with durable MagGraph memory.

## MagAgent vs Aider

Aider is a mature terminal pair programmer with strong Git-native editing and repo-map workflows. MagAgent is less mature as a pure coding loop, but broader: persistent memory, workbench records, recipes, playbooks, local UI, gateway integrations, and productivity tools.

Choose MagAgent when durable personal/project memory and repeatable workflows matter more than the fastest terminal edit loop.

## MagAgent vs Cline

Cline is strongest inside editors, especially with approval-gated file edits, terminal commands, MCP, and browser automation. MagAgent is terminal-first and now has optional browser helpers, but it does not yet offer a comparable editor sidebar.

Choose MagAgent when you want a CLI-first tool that travels across editors and keeps local workbench state.

## MagAgent vs Goose

Goose is a broad open-source local agent with CLI, desktop, and API surfaces. MagAgent overlaps philosophically, but differentiates through MagGraph memory, explicit memory inbox review, project playbooks, recipe plans, patch queues, and release/checkpoint workflows.

Choose MagAgent when the durable knowledge graph and auditable local workflow ledgers are the main draw.

## MagAgent vs Codex

Codex has a larger product surface: CLI, IDE, cloud tasks, desktop supervision, parallel agents, and deep model integration. MagAgent is smaller and local-first, but model/provider-agnostic and transparent by default.

Choose MagAgent when you want open local workflows, inspectable files, and provider choice.
